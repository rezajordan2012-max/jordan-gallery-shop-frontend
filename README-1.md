# فرانت‌اند فروشگاه Jordan Gallery Shop

این پروژه شامل فروشگاه (با دسته‌بندی محصولات، سبد خرید، ورود/ثبت‌نام و پرداخت) و **پنل مدیریت** (برای افزودن/ویرایش/حذف محصولات) است.

## راه‌اندازی محلی (اختیاری، برای تست روی سیستم خودت)

```bash
npm install
npm run dev
```

## دیپلوی رایگان روی Vercel

1. یک ریپازیتوری گیت‌هاب جدید بساز، مثلاً `jordan-gallery-shop-frontend`
2. همه‌ی فایل‌های این پوشه را در آن آپلود کن (همان روش «choose your files» که برای بک‌اند استفاده کردی)
3. برو به [vercel.com](https://vercel.com) و با گیت‌هاب وارد شو
4. **Add New → Project** را بزن و ریپازیتوری `jordan-gallery-shop-frontend` را انتخاب کن
5. Vercel به‌صورت خودکار تشخیص می‌دهد که این یک پروژه Vite است؛ چیزی را تغییر نده و روی **Deploy** بزن
6. بعد از چند دقیقه، آدرسی مثل `https://jordan-gallery-shop-frontend.vercel.app` می‌گیری

## وصل کردن دامنه‌ی خودت (jordangalleryshop)

1. در داشبورد پروژه روی Vercel، برو به **Settings → Domains**
2. دامنه‌ای که خریدی را وارد کن
3. Vercel چند رکورد DNS (معمولاً یک CNAME یا A record) نشانت می‌دهد
4. این رکوردها را در پنل مدیریت دامنه‌ات (جایی که دامنه را خریدی) وارد کن
5. معمولاً بعد از چند ساعت (گاهی تا ۲۴ ساعت) دامنه فعال می‌شود

## تنظیم نهایی بک‌اند بعد از دیپلوی فرانت‌اند

بعد از این‌که آدرس نهایی فرانت‌اند را گرفتی (چه آدرس Vercel، چه دامنه‌ی خودت)، باید در Render این دو متغیر محیطی را آپدیت کنی:

- `FRONTEND_URL` → آدرس نهایی فرانت‌اند
- `CALLBACK_URL` → `آدرس بک‌اند + /payment/callback` (مثلاً `https://jordan-gallery-shop-backend.onrender.com/payment/callback`)

## نکته درباره‌ی پنل مدیریت

پنل مدیریت (دکمه‌ی «پنل مدیریت» بالای سایت) از حافظه‌ی مرورگر (localStorage) استفاده می‌کند. یعنی محصولاتی که اضافه/ویرایش می‌کنی، فقط روی همان مرورگری که با آن پنل را باز کرده‌ای ذخیره می‌مانند — نه برای همه‌ی بازدیدکنندگان سایت. برای این‌که تغییرات پنل مدیریت برای همه‌ی مشتری‌ها نمایش داده شود، باید مدیریت محصولات را به بک‌اند (همان سروری که ساختیم) وصل کنیم؛ اگر خواستی این کار را هم انجام می‌دهیم.
